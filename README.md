# llmbrary

Turn a physical bookshelf into a portable knowledge base for LLMs.

This repository contains a structured catalog of 219 books extracted from photographs of a personal library using vision-based identification. The catalog is designed to be loaded into LLM context windows, giving AI assistants awareness of the intellectual background behind a conversation — the books that shaped how you think, what traditions you draw on, and where your references come from.

## Why

LLMs are general-purpose by default. They don't know what you've read, what intellectual traditions matter to you, or what conceptual vocabulary you actually use. Giving a model access to your library doesn't make it an expert in those books — but it does let it recognize when a project intersects with something on your shelves, trace thematic connections you might not have articulated, and meet you closer to where you actually are.

## Catalog Format

Each entry in `catalog.json` contains:

| Field | Type | Description |
|---|---|---|
| `title` | string | Book title |
| `author` | string\|null | Author or editor |
| `year` | integer\|null | Publication year |
| `synopsis` | string | Description and intellectual context |
| `themes` | string[] | Key themes and subjects |
| `source_image` | string | Reference photograph filename |
| `confidence` | "high"\|"medium" | Identification confidence |
| `needs_review` | boolean | Whether entry needs verification |
| `in_conversation_with` | string[] | Related titles in the collection (max 5) |

The `in_conversation_with` field maps relationships between books — not citations, but thematic, methodological, or historical resonance. These relationships allow traversal of intellectual lineages across the collection.

The full schema is defined in `schema.json`.

## Using with LLMs

### Direct context injection (any LLM)

The simplest approach: paste `CONTEXT.md` into your system prompt or context window. At ~1,150 lines it fits comfortably in most modern context windows.

**Claude (Projects or system prompt):**
Add `CONTEXT.md` as a project knowledge file, or paste it into a system prompt.

**ChatGPT / GPT-4:**
Paste `CONTEXT.md` into the system message, or upload it as a file in a conversation.

**Other models:**
Any LLM that accepts a system prompt or document context can use `CONTEXT.md` directly.

### Structured queries

Load `catalog.json` for programmatic access — filtering by theme, author, year, or traversing the relationship graph. Useful for building tools on top of the catalog or for selective context loading when the full file is too large.

### Claude skill integration

The `claude/SKILL.md` file is a skill wrapper for Claude Code and Cowork. It instructs Claude on how to load and use the library context: surfacing relevant books during projects, tracing intellectual lineages through the relationship graph, and recognizing when work touches themes present in the collection.

To use it, add the `claude/` directory to your Claude Code or Cowork skills path.

## Extraction Methodology

The catalog was built through a multi-stage vision pipeline:

1. **Photography** — 37 photographs of bookshelves capturing spine text at readable resolution
2. **Vision extraction** — LLM vision models identified titles and authors from spine images, producing `extracted_titles.json`
3. **Enrichment** — Each identified title was enriched with publication year, synopsis, themes, and confidence scoring
4. **Relationship mapping** — `in_conversation_with` links were generated by analyzing thematic overlap across the full collection
5. **Context generation** — `scripts/regenerate.py` renders the structured catalog into the markdown format in `CONTEXT.md`, organized into 30 thematic categories

Some books could not be identified from photographs (recorded in `unreadable.json`). Entries with `confidence: "medium"` or `needs_review: true` may contain inaccuracies.

## Workflow

Adding new books follows a three-stage pipeline: ingest photos, extract and merge titles, then regenerate the context file.

### 1. Drop photos

Place new bookshelf photographs in `photos/inbox/`. Any `.jpg`, `.jpeg`, `.png`, or `.heic` files will be picked up.

### 2. Run ingestion

```bash
# Preview what's new
python3 scripts/ingest.py --dry-run

# Log new images and produce inbox_manifest.json
python3 scripts/ingest.py

# After extraction, move processed images out of inbox
python3 scripts/ingest.py --move
```

The script writes `processing_log.json` so images are never processed twice.

### 3. Extract titles

Use your preferred vision extraction method (LLM vision, manual, etc.) to produce a `new_extractions.json` file — an array of objects following `schema.json`. At minimum each entry needs a `title`; missing enrichment fields will be flagged for review.

### 4. Merge into catalog

```bash
# Preview the merge
python3 scripts/merge_catalog.py new_extractions.json --dry-run

# Merge for real
python3 scripts/merge_catalog.py new_extractions.json
```

The merge script deduplicates by fuzzy title matching (configurable threshold via `--threshold`) and flags entries that need enrichment (synopsis, themes, etc.) with `needs_review: true`.

### 5. Regenerate CONTEXT.md

```bash
python3 scripts/regenerate.py
```

This rebuilds `CONTEXT.md` from the current state of `catalog.json`, preserving thematic groupings.

## Contributing

To add books manually, append entries to `catalog.json` following the schema in `schema.json`. Then regenerate the context file:

```bash
python3 scripts/regenerate.py
```

When adding entries:

- Set `confidence` to `"high"` for manually verified entries
- Include `in_conversation_with` references to existing titles where thematic connections exist (max 5)
- Write synopses that capture intellectual context, not just plot summary
- Choose themes that reflect how the book functions in a broader intellectual landscape
